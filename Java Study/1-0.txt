CRUD　　　　　　　　　　　　　　Create（生成）、Read（読み取り）、Update（更新）、Delete（削除）ユーザインタフェースが備える
　　　　　　　　　　　　　　　　べき機能
トランザクション                結果の整合性が要求される、複数の処理を1つにまとめたもの。                
  コミット                        処理や変更などを確定させる、反映させる、といった意味  
  ロールバック                    データ更新などで障害が起こったときに、その前の状態にまで戻ることをいう

排他制御（排他ロック）          複数の主体が同じ資源を同時に利用すると競合状態（race condition）が生じる場合に、ある主体が
　　　　　　　　　　　　　　　　資源を利用している間、別の主体による資源の利用を制限もしくは禁止する仕組みのこと
  楽観ロック                      めったなことでは他者との同時更新は起きないであろう、という楽観的な前提の排他制御
  悲観ロック　　　　　　　　　　　他者が同じデータに頻繁に変更を加えるであろう、という悲観的な前提の排他制御

■ 各種プロトコル
HTTP                            不特定多数と通信するときに使う通信プロトコルのひとつ　(通信の内容が暗号化されない）
FTP　　　　　　　　　　　　　　 特定ホスト間でファイルを「転送」する目的で作られたプロトコル
HTTPS　　　　　　　　　　　　　 不特定多数と通信するときに使う通信プロトコルのひとつ　(通信の内容が暗号化される）
SSL　　　　　　　　　　　　　　 インターネット上でやり取りする情報を暗号化して送受信するための仕組み
SSH                             ネットワークを経由して他のコンピュータに接続し、遠隔操作するための仕組み(通信の内容が暗号化される）
ポート                          ネットワークとパソコンの間にある接続口
ポート番号                      ポートに割り振られている識別用の番号のこと
HTTP、FTP、SSHの各ポート番号    TCP 80 : HTTP  TCP 20 : FTP (データ)  TCP 21 : FTP (制御)  TCP 22 : SSH
ファイアウォール                ネットワークの門番
TCP                             信頼性は高いが転送速度が低いという特徴がある通信プロトコルのひとつ
